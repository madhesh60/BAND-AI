# CLI Usage Guide

The Code Assistant is controlled exclusively via a CLI interface. You can run one-off commands or launch an interactive shell.

---

## 1. Subcommands

Execute tasks directly from your terminal using `uv run code-assistant`:

### Start a Task
Starts a new multi-agent development pipeline for the specified task.
```bash
uv run code-assistant run "Implement a fibonacci calculator in math_utils.py"
```

### List Agents
Lists the active agent registry, their roles, and system descriptions.
```bash
uv run code-assistant agents
```

### View Status
Shows status details of the latest active pipeline.
```bash
uv run code-assistant status
```

### History
Lists all execution logs or shows task breakdowns for a specific pipeline ID.
```bash
# List all past pipelines
uv run code-assistant history

# View subtasks and statuses for a specific pipeline
uv run code-assistant history <pipeline-id>
```

### Show Code Changes
Shows file diffs and contents generated by a pipeline.
```bash
# Show all code changes for the latest pipeline
uv run code-assistant show

# Show code changes for a specific pipeline ID
uv run code-assistant show --pipeline-id <pipeline-id>

# Filter by file path
uv run code-assistant show --file "math_utils.py"
```

### Approve Step
Approves a paused plan approval or code approval step, resuming the workflow.
```bash
uv run code-assistant approve <pipeline-id>
```

### Reject Step
Rejects a paused plan or code approval step, providing feedback to guide the agents.
```bash
uv run code-assistant reject "Please add input validation for n < 0" --pipeline-id <pipeline-id>
```

---

## 2. Interactive Click REPL Shell

To start the interactive shell, run `uv run code-assistant` without any subcommands:
```bash
uv run code-assistant
```

Inside the REPL, use the following slash commands:

### `/task <description>`
Launches a new coding pipeline task.
* **Example**: `> /task Create a date parser utility`

### `/status`
Shows the status table for the latest pipeline, including task completion rate and review verdicts.

### `/history [id]`
Lists all pipelines, or shows the subtask checklist for a specific pipeline.
* **Example**: `> /history`
* **Example**: `> /history a5b8a1c9-7d0e-4f32-8a9d-1b2c3d4e5f6g`

### `/show [id]`
Displays syntax-highlighted code changes for the pipeline.
* **Example**: `> /show`

### `/approve [id]`
Approves the current implementation plan or code changes and resumes execution.
* **Example**: `> /approve`

### `/reject [id] <feedback>`
Rejects the current plan/code and passes feedback comments to the planning or debugging agents.
* **Example**: `> /reject Add tests for negative numbers`

### `/help`
Prints the help screen displaying all available commands.

### `/exit` or `/quit`
Exits the interactive shell.
